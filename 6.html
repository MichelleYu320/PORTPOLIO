프로젝트 개요

AI 디지털교과서(AIDT) 콘텐츠 플랫폼에서 기존의 단순 파일/DB 이관 방식을 구조화된 선택 배포 시스템으로 전환하는 프로젝트를 주도했습니다. 브랜드, 콘텐츠 유형, 특정 콘텐츠별 선택적 배포가 가능한 시스템을 구축하고, 데이터 정합성 관리 체계를 확립했습니다.

기존 시스템의 한계

타 브랜드의 콘텐츠를 마이그레이션하는 과정에서 기존의 단순 파일/DB 이관 방식의 한계를 발견했습니다. HWP 파일을 HTML로 변환하는 과정에서 다양한 규칙과 메타데이터를 맞추는 작업이 복잡하고 비효율적이었습니다.

  • 일괄 이관으로 인한 불필요한 데이터 포함
  • 브랜드별, 콘텐츠별 선택적 배포 불가
  • 메타데이터 불일치로 인한 품질 문제
  • 수동 검수 작업으로 인한 시간 소요

주요 개선 사항

선택적 배포 시스템

브랜드, 콘텐츠 유형, 특정 콘텐츠별로 필요한 데이터만 선택하여 배포할 수 있는 구조화된 시스템을 설계했습니다.

  • 브랜드별 콘텐츠 분류 체계 구축
  • 콘텐츠 유형별 배포 정책 수립
  • 세분화된 선택 기준 정의

데이터 정합성 시스템

데이터 정합성의 필요성을 인식하고 자동화된 검증 및 관리 시스템을 개발했습니다.

  • 실시간 데이터 무결성 검증
  • 메타데이터 일관성 자동 검사
  • 배포 전후 데이터 대조 시스템

메타데이터 표준화

HWP에서 HTML 변환 과정의 복잡성을 해결하기 위한 메타데이터 표준화 작업을 수행했습니다.

  • 문서 형식 변환 규칙 정의
  • 메타데이터 매핑 테이블 구축
  • 버전 관리 및 이력 추적

개발자 협업 강화

개발자와의 밀접한 소통을 통해 메타데이터 및 DB 구조에 대한 이해도를 크게 향상시켰습니다.

  • DB 스키마 설계 참여
  • API 명세서 공동 작성
  • 데이터 모델링 협업

배운 점과 인사이트

기획자의 기술적 이해의 중요성

단순히 요구사항을 전달하는 것을 넘어 DB 구조와 메타데이터를 이해함으로써 개발자와 더 효과적으로 소통하고 현실적인 솔루션을 제안할 수 있었습니다.

데이터 품질의 근본적 중요성

아무리 좋은 시스템을 구축해도 데이터 자체의 품질이 보장되지 않으면 전체 서비스의 신뢰성이 떨어진다는 것을 경험하며 데이터 정합성의 중요성을 깨달았습니다.

점진적 개선의 효과

기존 시스템을 완전히 교체하기보다는 단계적으로 개선해 나가는 접근이 리스크를 줄이고 지속적인 발전을 가능하게 한다는 것을 배웠습니다.
